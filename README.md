Общее описание

Данный проект представляет собой проект на языке C, реализующий четыре алгоритма сортировки: 
1) Быструю сортировку
2) Сортировку слиянием
3) Внешнюю сортировку с буферами
4) Параллельную сортировку слиянием.

Проект включает полный набор автоматических тестов, примеры практического применения в реальных сценариях и 
детальное сравнение производительности всех алгоритмов. 
Основное внимание уделено внешней сортировке, которая позволяет обрабатывать данные, 
не помещающиеся в оперативную память, и параллельной сортировке для ускорения на многоядерных системах.

Теоретическая часть

1. Быстрая сортировка (Quicksort).

Быстрая сортировка является одним из наиболее эффективных алгоритмов сортировки. 
Алгоритм работает следующим образом: выбирается опорный элемент, массив разбивается на две части таким образом, 
что все элементы меньше опорного оказываются слева, а все элементы больше опорного — справа, 
после чего каждая часть рекурсивно сортируется. В данной реализации опорный элемент выбирается как средний элемент массива, 
а разбиение выполняется с использованием двух указателей, движущихся навстречу друг другу. 
Такой подход гарантирует хорошую производительность даже на уже отсортированных данных. 
Средняя сложность алгоритма составляет O(n log n), однако в худшем случае может достигать O(n²), 
хотя выбор среднего элемента значительно снижает вероятность деградации.

2. Сортировка слиянием (Merge Sort)

Сортировка слиянием представляет собой устойчивый алгоритм. 
Массив рекурсивно делится на две равные половины до тех пор, пока в каждой части не останется по одному элементу, 
после чего выполняется слияние двух отсортированных половин в один отсортированный массив.
Основное преимущество этого алгоритма заключается в гарантированной сложности O(n log n) во всех случаях, 
независимо от исходного состояния данных. Кроме того, сортировка слиянием является устойчивой, 
то есть сохраняет относительный порядок равных элементов. 
Основной недостаток — необходимость выделения дополнительной памяти O(n) для хранения временного массива при слиянии.

3. Внешняя сортировка слиянием (External Merge Sort)

Внешняя сортировка слиянием применяется в ситуациях, когда объём данных превышает доступную оперативную память и данные хранятся на диске. 
Алгоритм состоит из двух основных фаз. 

Первая фаза — сортировка сегментов: данные разбиваются на сегменты, размер которых помещается в оперативную память, 
каждый сегмент сортируется в памяти с использованием быстрой сортировки, 
после чего отсортированные сегменты записываются на диск в виде временных файлов. 

Вторая фаза — k-путевое слияние: все сегменты сливаются в один отсортированный файл. 

Особенность данной реализации заключается в использовании буферов фиксированного размера, 
что позволяет не загружать все сегменты в память целиком, а читать их по частям. 
Это делает алгоритм эффективным для работы с данными любого размера. 

4. Параллельная сортировка слиянием (Parallel Merge Sort)

Параллельная сортировка слиянием использует многопоточность для ускорения обработки данных на многоядерных системах. 
Принцип работы заключается в рекурсивном разделении массива на части, 
каждая из которых сортируется в отдельном потоке, а затем выполняется слияние отсортированных частей.
Глубина рекурсии ограничивается для предотвращения создания чрезмерного количества потоков. 
Ключевая оптимизация заключается в выделении одного временного массива на всю сортировку, 
что позволяет избежать многократных вызовов malloc() и free() внутри каждого слияния, 
которые создавали бы глобальную блокировку и замедляли работу. 
Такая реализация обеспечивает значительное ускорение на больших массивах данных.

Структура проекта

sorting.h — заголовочный файл, содержащий объявления всех публичных функций: 
быстрая сортировка, сортировка слиянием, внешняя сортировка с буферами, параллельная сортировка слиянием, 
а также вспомогательные функции для тестов.

sorting.cpp — основной файл с реализацией всех алгоритмов. 
Содержит четыре основных раздела: 
1. быстрая сортировка с улучшенным разбиением,
2. сортировка слиянием с оптимизированным использованием памяти
3. внешняя сортировка с буферами
4. параллельная сортировка слиянием с оптимизацией выделения памяти.

ProjectSort1.cpp — файл с тестами и примерами применения. 

Включает четыре группы автоматических тестов для каждого алгоритма, 
пять примеров практического использования и сравнение производительности всех алгоритмов.

Тесты

Проект включает четыре группы автоматических тестов и пять примеров применения. 
Тесты проверяют корректность работы алгоритмов на различных типах данных: 
случайные массивы, массивы с одинаковыми элементами, уже отсортированные массивы, 
обратно отсортированные массивы, массивы с отрицательными числами, массивы из одного элемента и пустые массивы. 
Каждый тест выводит массив до и после сортировки для наглядной демонстрации работы алгоритма.

Примеры применения

В проекте реализовано пять примеров практического использования алгоритмов сортировки. 

Первый пример демонстрирует внешнюю сортировку базы данных с десятью тысячами записей, показывая, 
как алгоритм эффективно обрабатывает большие объёмы данных. 

Второй пример показывает параллельную обработку двухсот тысяч элементов с использованием нескольких потоков. 

Третий пример демонстрирует гибридный подход: сортировку логов сервера с параллельной обработкой сегментов и внешним слиянием. 

Четвёртый пример показывает сортировку оценок студентов с вычислением статистических показателей. 

Пятый пример демонстрирует полный цикл работы с файлами: чтение данных с диска, сортировку и запись результата обратно на диск.
